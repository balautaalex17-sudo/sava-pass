## Notes / findings

- **Orphan files (not loaded by the homepage):** `step-event.png` (5.6 MB), `step-pay.png` (3.4 MB), `step-qr.png` (3.6 MB), `savapass-disintegrate.mp4` (4.8 MB). ~17 MB of dead weight in `web/public/imersiv/` — safe to delete.
- Videos autoplay, so `preload="metadata"` does not prevent the full download. Re-encoding bitrate/resolution is the lever (background loops tolerate modest quality).
- `event-easter.png` / `event-cupid.png` are eager despite being below the fold → make lazy.
- Middleware (`proxy.ts`) is scoped to auth routes only → not a homepage cost.
- Data layer: homepage does a live `getActiveEvent` query each load (capped 2 s); no caching (addressed in U5).

## After the pass (U1-U6) — same measurement

| Metric | Baseline | After | Change |
| --- | --- | --- | --- |
| Initial transfer | 13.49 MB | **1.85 MB** | −86% |
| Full transfer | 29.04 MB | **2.60 MB** | −91% |
| Image bytes | 20.68 MB | 0.99 MB | −95% |
| Video bytes | 7.40 MB | 0.66 MB | −91% |
| LCP (localhost) | 1016 ms | **460 ms** | −55% |
| Load → networkidle | 11.9 s | 8.4 s | −29% |
| Homepage rendering | dynamic (DB/load) | **Static / ISR 5m** | no per-load DB |
| Engine libs | serial unpkg.com | self-hosted + preloaded | no external hop |
| Touch scroll | Lenis JS-driven | native + parallax gated off | jank removed |
| DB cold-hang | ~5 s on pause | daily keep-warm cron | prevented |

What changed: U2 media re-encode (WebP + compressed video, −17 MB dead files deleted),
U3 self-hosted/preloaded engine libs, U4 keep-warm cron, U5 cached event reads +
homepage ISR (buyer pages stay dynamic for live stats), U6 gated the perpetual QR
box-shadow loop off touch. Regression check: touch st=0 / desktop st=9, reveals +
hero reveal fire on both, 0 console errors.
